# 虚幻引擎智能学习助手 — 开始使用

## 不懂虚幻引擎也没关系

**本助手面向零基础用户**：你不需要会编程、不需要用过 3D 软件，只要想学虚幻引擎（UE），就可以通过**智能问答**的方式学习。

## 你可以这样用

- **直接提问**：例如「虚幻引擎是什么」「怎么安装」「界面里这个按钮是干什么的」「下一步怎么做」。
- **发截图问**：可以**发送 UE 界面截图**，问「这个界面怎么用」「这个按钮是什么」「这一步该点哪里」，助手会结合截图用**菜单/面板/按钮**的步骤说明，方便对着界面操作。
- **看流程图**：助手在讲安装、创建项目、蓝图流程等步骤时，会给出 **Mermaid 流程图**；你可把代码块复制到 [Mermaid Live](https://mermaid.live) 查看图示。
- **看推荐视频**：回答里会视情况推荐与问题相关的**视频链接**（官方或 B 站等），便于跟着视频学。
- **随时追问**：如果没听懂，可以说「能再讲简单点吗」「具体点怎么做」，助手会换一种方式解释。
- **按课程学**：助手会结合虚拟现实 UE 课程教案和知识库回答，必要时会拉取 Epic 官方最新学习资源。

## 建议的问法示例

- 我想从零开始学 UE，该从哪里入手？
- UE 课程的教学目标是什么？
- 什么是蓝图？什么是 Lumen？
- 如何创建第一个 UE 项目？
- 官方教程和资源在哪里？
- （发一张 UE 界面截图）这个窗口是干什么的？这个按钮怎么用？

启动 nanobot 后，在对应通道（如 Telegram、命令行等）中直接输入你的问题即可开始学习。

## 本机命令行运行时若出现「未返回最终回复」

若在本机执行 `nanobot agent -m "UE 课程的教学目标是什么？"` 时出现「已处理完成，但模型未返回最终回复」或 “no response to give”，多半是**本机使用的 workspace 里没有教案**（默认 workspace 是 `~/.nanobot/workspace`，不是项目里的 `workspace` 文件夹）。

**解决办法**：让 nanobot 使用本项目自带的教案与知识库，把 workspace 指到项目下的 `workspace` 目录。

1. 打开或创建配置文件（Windows 一般在 `C:\Users\你的用户名\.nanobot\config.json`）。
2. 在配置里设置 `agents.defaults.workspace` 为**本机**上该项目的 `workspace` 文件夹的**绝对路径**。  
   例如本项目在 `D:\111科研\2026挑战杯\nanobot` 时，可写成：
   ```json
   {
     "agents": {
       "defaults": {
         "workspace": "D:/111科研/2026挑战杯/nanobot/workspace"
       }
     },
     "tools": {
       "restrictToWorkspace": true
     }
   }
   ```
   `restrictToWorkspace: true` 会让教案路径（如 `curriculum/index.json`）相对上述 workspace 解析，推荐一起打开。
3. 保存后重新执行 `nanobot agent -m "UE 课程的教学目标是什么？"` 再试。
