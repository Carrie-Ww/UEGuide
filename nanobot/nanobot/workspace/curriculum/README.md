# 教案管理系统使用说明

## 概述

教案管理系统是nanobot的一个技能模块，用于管理教学计划和基于教案内容的智能问答。

## 目录结构

```
workspace/
└── curriculum/
    ├── index.json              # 教案索引文件（包含所有教案的元数据）
    ├── lessons/                # 教案文件存储目录
    │   ├── lesson-001.md       # 具体教案文件
    │   └── example-template.md # 教案模板
    └── README.md               # 本文件
```

## 快速开始

### 1. 导入教案

当你有教案文件时，可以这样导入：

```
用户：导入教案文件"虚拟现实UE课程某职业技术学院教案表.md"
```

系统会自动：
- 读取教案文件内容
- 解析教案信息（课程名称、课时等）
- 生成唯一教案ID
- 保存到 `curriculum/lessons/lesson-{id}.md`
- 更新 `curriculum/index.json` 索引

### 2. 查询教案

**列出所有教案**：
```
用户：列出所有教案
```

**搜索特定教案**：
```
用户：查询UE相关的教案
用户：查找包含"虚拟现实"的教案
```

### 3. 智能问答

基于教案内容回答问题：

```
用户：UE课程的教学目标是什么？
用户：如何创建UE项目？
用户：这门课程有多少课时？
```

系统会：
1. 从问题中提取关键词
2. 在索引中搜索相关教案
3. 读取匹配的教案内容
4. 基于教案内容生成准确回答

## 索引文件格式

`index.json` 文件结构：

```json
{
  "lessons": [
    {
      "id": "ue-course-001",
      "title": "虚拟现实UE课程",
      "course": "虚拟现实UE",
      "hours": 64,
      "created_at": "2026-02-09T10:00:00",
      "updated_at": "2026-02-09T10:00:00",
      "file_path": "lessons/lesson-ue-course-001.md",
      "keywords": ["UE", "虚拟现实", "Unreal Engine", "游戏开发"]
    }
  ],
  "version": "1.0",
  "created_at": "2026-02-09T00:00:00"
}
```

## 教案文件格式

教案文件使用Markdown格式，建议包含以下部分：

- 课程基本信息（名称、代码、课时等）
- 课程教学目标
- 教学内容（按章节组织）
- 教学方法
- 教学资源
- 考核方式
- 教学进度安排

参考 `lessons/example-template.md` 查看完整模板。

## 使用技巧

1. **关键词提取**：系统会自动从教案标题和内容中提取关键词，便于搜索
2. **多教案支持**：可以导入多个教案，系统会维护统一的索引
3. **智能匹配**：问答时会自动匹配最相关的教案内容
4. **内容更新**：可以随时更新教案内容，系统会同步更新索引

## 注意事项

- 教案文件使用UTF-8编码
- 建议使用Markdown格式，保持结构清晰
- 索引文件会自动维护，不要手动修改
- 删除教案时，记得同时更新索引文件
